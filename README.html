ProteinEvolver 2.2.1 README

21 May 2025


Purpose

ProteinEvolver simulates protein evolution along given –phylogenetic tree– or simulated phylogenetic histories –including population birth-death and coalescent (with complex heterogeneous recombination rate, migration and demographics) phylogenetic processes– under empirical and structurally constrained substitution models.
A relevant new feature is that the simulated evolutionary history can be affected by the fitness (protein folding stability) of the simulated protein sequences by combining the population birth-death population model (that simulates the evolutionary history) with a structurally constrained substitution model (i.e., protein variants with a high fitness lead to high birth and low death rates, resulting in a high number of descendants; and the opposite for protein variants with a low fitness that may lead to low birth and high death rates resulting in extinctions). Thus, it simulates both the evolutionary history and molecular evolution simulatenously, by applying selection from the folding stability in every molecular variant to simulate every node of the phylogenetic tree and protein evolution.
The file ProteinEvolver2.2.1.manual.pdf includes detailed documentation, and the user is strongly advised to read it carefully before using the program.


Download and Installation

Download the program fom https://github.com/MiguelArenas/proteinevolver. Open a console window. In the same directory as the file ProteinEvolver2.2.1.zip, type:

unzip ProteinEvolver2.2.1.zip

You should now see a folder called ProteinEvolver-2.2.1. Note that this zip file was made on a mac, so in windows or linux you might see some strange files, that start with "." or "-". These are macintosh hidden files, which can be safely removed without affecting the software.

Executables for MacOSX and Linux are available inside the folder exe.


Compilation

The program should compile without much problem in any Unix-like environment (Linux, MacOSX, etc.). A makefile is provided for the gcc compiler, but if you have cc instead, just change gcc for cc in the makefile. The default optimization level is set to -O3. To compile in macintosh with an intel processor, and probably in othe machines, you can enable the option -fast instead.

To compile the program move into the folder src inside the ProteinEvolver-2.2.1 folder and type:

make all

to compile the executable in the current folder (src) inside the ProteinEvolver-2.2.1 folder, or

make install

to clean, compile and install the new executable in the exe folder inside the ProteinEvolver-2.2.1 folder.

The program should compile without problems with a second makefile "Makefile_MPI" for a MPI version. This makefile might need some modifications for particular OS. To compile the program type:

make -f Makefile_MPI


Execution

 

The program runs in a command-line window. In MacOS X and Linux the user needs to open the Terminal application, and in Windows XP the user

should open the Command Prompt. To run the program move to the folder where the executable is located and type:


./ProteinEvolver2.2.1



For execute the parallel version, MPI libraries have to be installed in the host. The minimum number of processors is two. An example of execution in 3 processors is the next:

mpirun -np 3 ProteinEvolver2.2.1


Arguments can be entered in the command line or, more conveniently, from a text file called "parameters" that should be located in the same directory as the executable. See the documentation for further details.
